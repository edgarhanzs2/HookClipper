Technical Documentation

v2.0 · March 2026

1. System Overview

AI Hook Clipper is a full-stack web application that uses artificial intelligence to automatically identify the most engaging "hook" moments in video content and generate short-form clips optimized for social media platforms such as TikTok, Instagram Reels, and YouTube Shorts.

The application supports multiple AI providers (OpenAI GPT-4o, Google Gemini 1.5 Flash, and local Ollama models) for transcription and hook detection. It features asynchronous job processing, YouTube URL ingestion, intelligent vertical cropping with face tracking, and dynamic subtitle generation.

Key Capabilities

  • Video upload and YouTube URL ingestion
  • AI-powered audio transcription (OpenAI Whisper, Gemini, mlx-whisper)
  • AI-powered hook/viral moment detection (GPT-4o, Gemini, Ollama)
  • Automatic clip trimming with FFmpeg
  • 9:16 vertical video cropping with face tracking via MediaPipe
  • Dynamic subtitle generation with multiple styles (Hormozi, Word Pop, Classic)
  • Asynchronous task processing with Celery and Redis
  • Re-trimming clips with custom timestamps
  • Mock AI mode for development and testing

2. Architecture

AI Hook Clipper follows a decoupled client-server architecture with an asynchronous processing layer.

System Layers

🖥️ Frontend — Next.js 14

React-based single-page application providing upload UI, YouTube URL input, real-time progress tracking, clip preview, timeline editing, and download functionality.

⚡ Backend API — FastAPI

RESTful API server handling HTTP requests, file management, job orchestration, and serving processed clips.

🔄 Worker Layer — Celery + Redis

Asynchronous task queue processing the full video pipeline: audio extraction → transcription → hook detection → clip trimming.

Data Flow

1

User uploads a video file or submits a YouTube URL via the frontend.

2

Frontend sends the request to the FastAPI backend.

3

Backend saves the file (or downloads from YouTube) and dispatches a Celery task.

4

Celery worker processes the video: Extract audio → Transcribe → Detect hooks → Trim clips.

5

Frontend polls the job status endpoint until processing completes.

6

User previews clips, optionally renders vertical (9:16) versions with subtitles.

7

User downloads individual clips.

4. Backend API Reference

Base URL: http://localhost:8000

GET /

Health check. Returns API status and version.

{ "status": "ok", "app": "AI Hook Clipper API", "version": "2.0.0" }

POST /upload

Upload a video file and start async processing. Accepts multipart form data.

Form Parameters

ParameterTypeRequiredDescription
fileFileYesVideo file (MP4, MOV, AVI, MKV, WebM)
mock_aistringNo"true" or "false" (default: "false")
providerstringNo"openai", "gemini", or "ollama" (default: "openai")
{ "job_id": "uuid", "status": "processing", "message": "Video uploaded. Processing started." }

POST /ingest

Download a YouTube video by URL and start async processing.

Request Body (JSON)

{ "url": "https://youtube.com/watch?v=...", "mock_ai": "false", "provider": "openai" }

Response

{ "job_id": "uuid", "status": "processing", "message": "YouTube video downloaded. Processing started." }

GET /status/{job_id}

Get the current processing status of a job. Returns step number and status label.

{ "job_id": "...", "step": 2, "status": "transcribing" }

GET /results/{job_id}

Get the processed clips for a completed job. Returns clip metadata including titles, timestamps, scores, and download URLs.

{
  "job_id": "...",
  "status": "completed",
  "clips": [
    {
      "title": "Viral Hook 1",
      "start_time": 12.5, "end_time": 45.0,
      "score": 95,
      "download_url": "/download/job_id/1"
    }
  ]
}

POST /retrim/{job_id}/{clip_id}

Re-trim a specific clip with new custom start/end timestamps.

Request Body

{ "start_time": 10.5, "end_time": 45.0 }

POST /render-vertical/{job_id}/{clip_id}?style=hormozi

Render a 9:16 vertical version of a clip with face-tracked cropping and subtitle overlay.

Query Parameter

ParameterOptionsDefault
stylehormozi, word_pop, classichormozi

GET /download/{job_id}/{clip_id}

Download a specific processed clip. Returns binary MP4 file.

GET /download-vertical/{job_id}/{clip_id}

Download the rendered vertical (9:16) version of a clip.

5. Service Modules

All service modules are located in backend/services/. Each module handles a specific stage of the processing pipeline.

audio.py — Audio Extraction

Extracts audio tracks from video files using FFmpeg for downstream transcription.

extract_audio(video_path, output_dir)Extracts audio from a video file and saves as MP3. Uses FFmpeg with 5-minute timeout.

transcription.py — OpenAI Transcription

Transcribes audio using OpenAI's Whisper API.

transcribe_audio(audio_path)Sends audio to OpenAI Whisper API and returns transcript with segments, timestamps, and full text.

transcription_gemini.py — Gemini Transcription

Transcribes audio using Google Gemini 1.5 Flash.

transcribe_audio(audio_path)Uploads audio to Gemini and returns transcript with word-level timestamps.

transcription_ollama.py — Local Transcription (mlx-whisper)

Transcribes audio using Apple Silicon optimized mlx-whisper for fully offline processing.

transcribe_audio(audio_path)Runs mlx-whisper locally. Model configurable via WHISPER_MODEL env var.

hooks.py — OpenAI Hook Detection

Uses GPT-4o to analyze transcripts and identify the most viral, engaging hook moments.

detect_hooks(transcript)Sends transcript to GPT-4o; returns top 3-5 hooks with scores, timestamps, and reasoning.

hooks_gemini.py — Gemini Hook Detection

Uses Gemini 1.5 Flash for hook detection with the same prompt strategy.

detect_hooks(transcript)Sends transcript to Gemini for viral moment detection.

hooks_ollama.py — Ollama Hook Detection (Local)

Uses local Ollama models (e.g., Llama 3.1) for fully offline hook detection with context window management.

detect_hooks(transcript)Sends transcript to local Ollama with auto-truncation. Model configurable via OLLAMA_MODEL.

trimmer.py — Video Trimming

Handles all clip trimming, including the full vertical render pipeline.

trim_clip(video_path, start_time, end_time, output_path)Trims a single clip from a video using FFmpeg with H.264/AAC codecs.
trim_all_clips(video_path, hooks, output_dir, job_id)Iterates over all detected hooks and trims each into an individual clip file.
trim_and_render_vertical(video_path, start, end, face_data, sub_path, out)Full pipeline: trim → crop 9:16 → burn subtitles in one FFmpeg pass.

crop_engine.py — Vertical Cropping

Crops landscape (16:9) video to portrait (9:16) format using face tracking coordinates.

crop_to_vertical(video_path, face_data, output_path, ...)Crops using average face position for stable, centered framing.
center_crop_vertical(video_path, output_path, ...)Fallback center crop when face tracking data is not available.

face_tracker.py — Face Tracking

Detects and tracks the dominant face using MediaPipe for intelligent vertical cropping.

track_faces(video_path, sample_every_n=5)Runs face detection on sampled frames; returns smoothed coordinates (0-1) with moving-average filter.

subtitle_engine.py — Subtitle Generation

Generates Advanced SubStation Alpha (.ass) subtitle files with multiple visual styles.

generate_subtitles(transcript, style, output_path, clip_start, clip_end)Creates .ass file from word-level transcript. Supports hormozi, word_pop, and classic styles.

downloader.py — YouTube Downloader

Downloads videos from YouTube URLs using yt-dlp.

validate_url(url)Validates against supported YouTube URL patterns.
get_video_info(url)Fetches metadata (title, duration, uploader) without downloading.
download_youtube(url, output_dir)Downloads the video at up to 720p as MP4.

6. Asynchronous Processing (Celery)

Video processing is handled asynchronously using Celery with Redis as the message broker and result backend. The API returns immediately with a job ID while processing continues in the background.

Configuration

File: backend/celery_app.py

REDIS_URL = os.getenv("REDIS_URL", "redis://localhost:6379/0")
celery = Celery("hook_clipper", broker=REDIS_URL, backend=REDIS_URL)

Running the Worker

# macOS recommended (avoids fork safety issues):
celery -A celery_app.celery worker --loglevel=info --pool=solo

9. Installation & Setup

Prerequisites

  • Python 3.13+
  • Node.js 18+ and npm
  • FFmpeg (brew install ffmpeg)
  • Redis server (brew install redis && redis-server)
  • yt-dlp (pip install yt-dlp or brew install yt-dlp)
  • (Optional) Ollama for local AI — ollama.ai

Backend Setup

cd backend
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# Copy and configure environment variables
cp .env.example .env
# Edit .env with your API keys

Frontend Setup

# From project root
npm install
npm run dev

Starting All Services

# Terminal 1: Redis
redis-server

# Terminal 2: Backend API
cd backend && source venv/bin/activate
uvicorn main:app --reload --port 8000

# Terminal 3: Celery Worker
cd backend && source venv/bin/activate
celery -A celery_app.celery worker --loglevel=info --pool=solo

# Terminal 4: Frontend
npm run dev

10. Deployment Guide

Production Considerations

  • Use a production ASGI server (gunicorn with uvicorn workers) instead of uvicorn --reload.
  • Configure Celery with a proper pool (prefork or gevent) on Linux servers.
  • Set up Redis with persistence if job state needs to survive restarts.
  • Serve the Next.js frontend via a CDN or static hosting (Vercel, Netlify).
  • Store uploaded videos and output clips on cloud storage (S3, GCS) instead of local disk.
  • Set appropriate CORS origins for the production frontend URL.
  • Use environment-specific .env files and never commit API keys to version control.
  • Set up log aggregation and monitoring for the backend and Celery workers.
